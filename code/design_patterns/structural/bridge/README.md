# Bridge Design Pattern in C++

简单概括一下桥接模式，两个关联但独立变化的东西不要使用继承来耦合在一起，而是通过组合的方式将它们连接起来，从而实现解耦和更好的扩展性。
在传统的继承结构中，如果抽象和实现高度耦合，增加新的实现或抽象会导致类爆炸（组合爆炸）。例如，有一个图形类层次结构和一个颜色类层次结构，如果每种图形都需要与每种颜色组合（绿色的圆、红色的矩形等乱七八糟的继承组合），就会产生大量的子类。而桥接模式的设计思路可以是将颜色类作为图形类的一个成员变量，通过组合的方式实现不同颜色和图形的组合，从而避免类爆炸问题。

GoF设计模式中，桥接模式通常包含以下几个角色：
- 抽象类（Abstraction）：定义了高层次的接口，并包含一个对实现部分的引用。
- 实现类接口（Implementor）：定义了实现部分的接口。
- 具体实现类（ConcreteImplementor）：实现了实现类接口，提供具体的实现。
- 扩展抽象类（RefinedAbstraction）：扩展了抽象类，通常包含更具体的操作。

感觉这几个概念有点抽象，没必要强行理解，反正核心就是“通过组合而不是继承来连接两个独立变化的维度”。
