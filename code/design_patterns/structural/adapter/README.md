# Adapter Design Pattern in C++

适配器模式核心思想在于将一个类的接口转换成客户端期望的另一个接口，让原本接口不兼容的类可以一起工作。
例如日常生活中的电源适配器，蓝牙适配器等，都是将一种接口转换为另一种接口的典型例子。
核心就是为了解决接口不兼容的问题。例如一个辅助驾驶系统需要从地图导航获取数据，原来用的是自研的导航地图API，现在想要改用高德地图API，但是高德地图的接口和原来自研的接口不兼容，这时就可以使用适配器模式，通过一个适配器类将高德地图的数据转换为自研接口所需的数据格式，从而使得辅助驾驶系统能够继续正常工作。甚至有可能需要同时支持多个不同的导航地图API，这时就可以为每个API实现一个适配器类或者使用更通用的适配器类来处理不同的API。

## 实现方式
- 类适配器：通过多重继承，将适配器类同时继承自目标接口和现有类，实现接口转换。
- 对象适配器：通过组合，将现有类作为适配器类的成员变量，在适配器类中实现目标接口的方法，并在这些方法中调用现有类的方法。
- 使用模板和泛型编程，可以实现更灵活的适配器，支持不同类型的现有类。
- 结合智能指针（如`std::shared_ptr`或`std::unique_ptr`）来管理适配器和被适配对象的生命周期，避免内存泄漏。
