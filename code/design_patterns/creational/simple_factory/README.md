# Simple Factory Design Pattern in C++

简单工厂模式是一种创建型设计模式，通过一个工厂类根据传入的参数决定创建哪一个具体类的实例。它将对象的创建逻辑封装在工厂类中，客户端只需调用工厂方法即可获得所需对象。注意，简单工厂模式并不是GoF设计模式中的正式模式。

它和工厂方法模式的区别在于，简单工厂模式通常只有一个工厂类（一个工厂 + switch语句或if-else语句来决定创建哪个产品），而工厂方法模式则允许多个工厂类，每个工厂类负责创建一种产品类型。

## 特点
- **封装创建逻辑**：将对象的创建过程封装在工厂类中，客户端无需了解具体的创建细节。
- **易于扩展**：添加新的产品类时，只需修改工厂类，无需修改客户端代码。
- **简化客户端代码**：客户端只需调用工厂方法，减少了代码重复。
- **集中管理**：所有产品的创建逻辑集中在一个地方，便于维护。
- **不符合开闭原则**：每次添加新产品类时都需要修改工厂类，违反了开闭原则。

## 适用场景
- 需要根据不同条件创建不同类型对象时。
- 客户端不需要了解对象创建的复杂过程时。
- 需要集中管理对象创建逻辑时。
- 需要简化客户端代码时。

## 注意事项
- 避免工厂类过于庞大，导致难以维护。
- 如果产品类较多且频繁变化，考虑使用工厂方法模式或抽象工厂模式。
- 确保工厂类的职责单一，专注于对象创建。
