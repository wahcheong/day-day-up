# Prototype Design Pattern in C++

原型模式是一种创建型设计模式，通过复制现有对象来创建新对象，而不是通过类的实例化。这样可以提高性能，尤其是在创建复杂对象时。

## 适用场景
- 对象的创建成本较高（需要大量计算、IO、数据库查询等），且需要频繁创建相似对象。
- 需要动态地创建对象，而不依赖于具体类。
- 希望通过复制现有对象来简化对象的创建过程。
- 希望隐藏对象创建细节，只暴露克隆接口。

## 实现方式
- 定义一个原型接口，包含一个克隆方法（如`Clone()`）。
- 具体原型类实现该接口，并在克隆方法中返回自身的副本。
- 客户端通过调用克隆方法来创建新对象，而不是直接实例化类。
- 原型注册表（可选）：维护一个原型对象的集合，允许客户端通过名称或类型获取并克隆原型对象。
