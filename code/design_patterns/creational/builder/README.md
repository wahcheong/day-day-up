# Builder Design Pattern

建造者模式（Builder Design Pattern）是一种创建型设计模式，旨在将一个复杂对象的构建过程与其表示分离，使得同样的构建过程可以创建不同的表示。通过使用建造者模式，可以更灵活地创建复杂对象，同时提高代码的可读性和可维护性。
精髓在于将对象的构建过程封装在一个独立的建造者类中，而不是直接在客户端代码中进行对象的创建。这样，客户端代码只需要与建造者接口交互，而不需要了解具体的构建细节。
现代C++中，建造者模式通常结合流式接口（Fluent Interface）使用，使得代码更加简洁和易读。通过链式调用，可以一步步地构建复杂对象，而不需要重复创建临时变量。

## 适用场景
- 需要构建的对象复杂，通常包含4个或更多的属性（参数）。
- 构建过程需要多个步骤，且这些步骤可以有不同的顺序或组合。
- 参数之间存在依赖关系，某些参数的设置可能影响其他参数的设置。
- 希望通过不同的建造者实现来创建不同类型的对象。
