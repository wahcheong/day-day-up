# Abstract Factory Method Design Pattern

抽象工厂方法设计模式是一种创建型设计模式，它提供了一种接口，用于创建相关或依赖对象的家族，而无需指定它们具体的类。通过使用抽象工厂方法，客户端代码可以独立于具体的产品类，从而提高了代码的灵活性和可维护性。
代码中使用的例子是为不同的平台（Windows和MacOS）创建UI组建。Windows平台的Button和Checkbox是同属于Windows产品家族了，通过抽象工厂方法，能够有效避免在Windows平台下创建MacOS的UI组件。

网上还看到一个很经典的抽象工厂例子：游戏中的不同种族（人族、精灵、兽人等）有不同的武器和防具。通过抽象工厂方法，可以为每个种族创建相应的武器和防具，而不需要在客户端代码中硬编码具体的类名。例如，人族使用剑和盾牌，精灵使用弓箭和轻甲，兽人使用斧头和重甲。也能够有效避免在创建武器时使用了错误的防具，例如为人族创建了兽人的防具。
